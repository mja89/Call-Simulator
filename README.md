# شبیه‌ساز تماس (Call Simulator) — پروژه نمونه اندروید

این یک پروژه‌ی Android Studio (Kotlin) است که یک محیط **شبیه‌سازی‌شده** برای صفحه تماس می‌سازد.
هیچ تماس واقعی گرفته نمی‌شود و به شبکه تلفن دسترسی ندارد؛ فقط رابط کاربری شبیه به صفحه تماس
داخل خود اپلیکیشن نمایش داده می‌شود.

## نحوه اجرا
1. پوشه `CallSimulator` را با Android Studio (Hedgehog یا جدیدتر) باز کنید.
2. اجازه دهید Gradle Sync انجام شود (نیاز به اینترنت برای دانلود وابستگی‌ها دارد).
3. روی یک دستگاه یا شبیه‌ساز اندروید (حداقل API 24) اجرا کنید.
4. هنگام اولین اجرا، مجوز دسترسی به مخاطبین (و در اندروید 13+، مجوز نوتیفیکیشن) را تایید کنید.

## ساختار و قابلیت‌ها

| بخش | فایل | توضیح |
|---|---|---|
| لیست مخاطبین | `MainActivity.kt` | مخاطبین واقعی گوشی را می‌خواند و نمایش می‌دهد |
| کانفیگ وویس هر مخاطب | `ContactConfigActivity.kt` | افزودن/حذف یک یا چند فایل صوتی برای هر مخاطب (از حافظه گوشی) |
| ذخیره‌سازی | `VoiceStorage.kt` | نگاشت مخاطب → لیست فایل صوتی در SharedPreferences (JSON) |
| تماس خروجی شبیه‌سازی‌شده | `CallActivity.kt` (TYPE_OUTGOING) | با زدن دکمه «تماس» روی مخاطب، صفحه تماس باز و وویس او پخش می‌شود |
| تنظیم تماس ورودی | `IncomingSimulatorActivity.kt` | انتخاب مخاطب مشخص یا «تصادفی» و تعیین تاخیر زمانی |
| اجرای تماس ورودی | `IncomingCallReceiver.kt` + `CallActivity.kt` (TYPE_INCOMING) | بعد از تاخیر تعیین‌شده، نوتیفیکیشن/صفحه تمام‌صفحه شبیه تماس ورودی باز می‌شود؛ با زدن «پاسخ» وویس مخاطب پخش می‌شود |

## نکات مهم برای توسعه بیشتر
- اگر مخاطبی هیچ فایل صوتی نداشته باشد، هنگام تماس فقط پیام هشدار نمایش داده می‌شود و صدایی پخش نمی‌شود؛
  بهتر است حتماً قبل از تست، از منوی «تنظیم وویس» برای چند مخاطب فایل صوتی اضافه کنید.
- برای اینکه صفحه تماس ورودی حتی وقتی گوشی قفل است هم دیده شود، از `setShowWhenLocked` +
  Full-Screen Notification استفاده شده؛ در برخی گوشی‌ها (مثل شیائومی/هواوی) ممکن است نیاز به فعال‌سازی
  دستی مجوز «نمایش روی صفحه قفل» یا خاموش کردن بهینه‌سازی باتری برای این اپ باشد.
- اگر خواستید حالت «زنگ خوردن» قبل از پاسخ هم صدا داشته باشد (شبیه رینگ‌تون واقعی)، کافیست در
  `CallActivity` برای حالت `TYPE_INCOMING` قبل از پاسخ، یک `Ringtone`/`MediaPlayer` جداگانه پخش کنید
  و با زدن دکمه پاسخ آن را متوقف و وویس اصلی مخاطب را استارت کنید.
- فایل‌های صوتی به‌صورت URI (از طریق Storage Access Framework) ذخیره می‌شوند، پس نیازی به مجوز
  Storage جداگانه نیست؛ اما اگر کاربر فایل را از مسیر انتخاب‌شده حذف/جابه‌جا کند، پخش آن دیگر کار نمی‌کند.
